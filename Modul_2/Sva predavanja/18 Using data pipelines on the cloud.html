Data pipelines su širi pojam, mogu obuhvaćati u sebi ETL¶

Koraci:¶

1.) Data ingestion - obuhvat podataka iz različitih izvora¶

2.) Data cleaning - čišćenje podataka od različitih grešaka¶

3.) Data transformation - pretvaranje podataka u odgovarući format¶

Ovo su dosad koraci ETL-a¶

4.) Data analysis - primjena različitih statističkih ili ML metoda kako bi se otkrile važne informacije u tim podacima¶

5.) Data visualisation - vizualizacija podataka u razumljivom formatu (grafikoni)¶

Izrada data pipelinea korištenjem AWS-a¶

Razumijevanje Komponenti¶

Prije nego što se upustimo u proces, važno je razumjeti ključne AWS usluge koje se često koriste u podatkovnim cjevovodima:¶

  • S3: Objektna pohrana za pohranu podataka.
  • Kinesis: Usluga za streaming podataka u stvarnom vremenu.
  • Lambda: Serverless usluga za pokretanje koda.
  • Glue: ETL usluga za transformaciju podataka.
  • Athena: Interaktivna usluga za upite podataka pohranjenih na S3.
  • EMR: Upravljana Hadoop i Spark usluga za obradu velikih podataka.

Uobičajene faze pri izradi data pipelinea (onih 5 gore) i odgovarajući AWS Alati¶

Prikupljanje Podataka¶

  • Alati: S3, Kinesis, Lambda
  • Proces: Podaci se prikupljaju iz različitih izvora (npr. baze podataka, API-jevi, IoT uređaji) i pohranjuju na S3 ili streamaju u Kinesis. Lambda funkcije mogu pokrenuti događaje za unos podataka.

Transformacija Podataka¶

  • Alati: Glue, Lambda, EMR
  • Proces: Podaci se čiste, normaliziraju i transformiraju u format prikladan za analizu. Glue se koristi za ETL zadatke, dok se Lambda ili EMR mogu koristiti za složenije transformacije.

Pohrana Podataka¶

  • Alati: S3
  • Proces: Transformirani podaci pohranjuju se na S3 za daljnju obradu ili analizu.

Analiza Podataka¶

  • Alati: Athena, EMR
  • Proces: Podaci se analiziraju korištenjem SQL upita (Athena) ili okvira za velike podatke poput Spark-a (EMR).

Vizualizacija Podataka¶

  • Alati: QuickSight, Tableau
  • Proces: Uvidi iz podataka vizualiziraju se korištenjem alata za vizualizaciju podataka.

Jezici i Vještine - Iako AWS usluge često nude sučelja koja smanjuju potrebu za direktnim programiranjem, znanje sljedećih jezika i vještina može biti korisno:¶

  • Python: Široko korišten za obradu i analizu podataka.
  • SQL: Za upite podataka u Atheni ili drugim bazama podataka.
  • Shell skripte: Za automatizaciju zadataka i upravljanje cjevovodima.
  • AWS CLI: Za interakciju s AWS uslugama iz naredbenog retka.
  • CloudFormation: Za definiranje i upravljanje AWS resursima kao kodom.

Primjer Data pipelinea: Obrada IoT Podataka u Stvarnom Vremenu¶

  • IoT uređaji generiraju podatke senzora i šalju ih u Kinesis.
  • Lambda funkcije obrađuju Kinesis stream, filtrirajući i transformirajući podatke.
  • Transformirani podaci pohranjuju se na S3.
  • Athena se koristi za upite podataka i generiranje analitike u stvarnom vremenu.
  • Rezultati se vizualiziraju u QuickSightu.

Primjer Data Pipelinea - kreiranje Data Warehousea i analiza podataka o prodaji¶

  • Data extraction: Glue / Lamda - kreiranje funkcija za učitavanje podataka iz csv fileova
  • Data transformation: Korištenjem Glue alata se podaci čiste i transformiraju (missing values, normalizacija podataka)
  • Data loading: Učitavanje podataka u S3
  • Data querying: korištenje Athene za postavljanje upita i izradu reporta

Primjer Data Pipelinea - izrada ML modela za predikciju churn ratea korisnika¶

Unos Podataka (Glue):¶

  • Uvoz podataka o korisnicima iz CSV datoteke ili baze podataka.
  • Čišćenje Podataka: Obrada nedostajućih vrijednosti, iznimke i neusklađenosti u podacima.
  • Inženjering Značajki: Kreiranje novih značajki ili transformacija postojećih kako bi se poboljšale performanse modela. ### Istraživanje Podataka (SageMaker Studio ili Jupyter Notebook):
  • Istraživanje podataka i razumijevanje odnosa između varijabli.
  • Vizualizacija podataka koristeći biblioteke poput Matplotlib ili Seaborn. ### Odabir Modela:
  • Odabir odgovarajućeg algoritma strojnog učenja na temelju prirode podataka i problema. Uobičajeni izbori za predviđanje odlaska korisnika (churn) uključuju logističku regresiju, random forest i gradient boosting. ### Treniranje Modela (SageMaker):
  • Korištenje ugrađenih algoritama SageMakera ili unos vlastitog algoritma
  • Trening modela na skupu podataka za treniranje koristeći odabrani algoritam.
  • Eksperimentiranje s različitim hiperparametrima kako bi se optimizirale performanse modela. ### Evaluacija Modela:
  • Procjena performansi modela na izdvojenom skupu podataka koristeći različite metrike ### Implementacija Modela:
  • Implementiranje treniranog modela kao SageMaker endpoint ili Lambda funkcija.
  • Korištenje implementiranog modela za predviđanja na novim podacima.

Code example za zadnji primjer (pseudokod)¶

import boto3 import sagemaker

Create a SageMaker session¶

sagemaker_session = boto3.session.Session()

Define the training job¶

  • training_job_name = 'customer-churn-model'
  • algorithm_name = 'xgboost' # Replace with your desired algorithm
  • role_arn = 'your_role_arn' # Replace with your IAM role ARN

Create a training job¶

  • training_job = sagemaker.TrainingJob(
    • name=training_job_name,
    • algorithm_name=algorithm_name,
      • role=role_arn, ... other parameters like instance count, instance type, etc. )

Start the training job¶

  • training_job.fit(training_input_path='s3://your-bucket/training-data', 
              - output_path='s3://your-bucket/model-output')

Deploy the model as an endpoint¶

  • predictor = training_job.deploy(
    • instance_type='ml.m4.xlarge',
    • initial_instance_count=1 )
